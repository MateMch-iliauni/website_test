01 — საჭიროება

პრობლემა, რომელსაც ვაგვარებთ

სახანძრო უსაფრთხოების საერთაშორისო კვლევების მიხედვით, საკვების მომზადება საცხოვრებელ შენობებში ხანძრების წამყვანი მიზეზია. NFPA-ის მონაცემებით, საყოფაცხოვრებო ხანძრების დაახლოებით 44% სწორედ სამზარეულო მოწყობილობებითაა გამოწვეული, ხოლო უყურადღებოდ დატოვებული გაზქურა დასახელებულია ხანძრისა და მსხვერპლის ნომერ პირველ ხელშემწყობ ფაქტორად.

არსებული უსაფრთხოების სისტემები — სტანდარტული კვამლის დეტექტორები — უკვე მომხდარ ფაქტზე რეაგირებენ. გაზქურაზე ალის ჩაქრობისას კვამლი არ წარმოიქმნება, გამოიყოფა მხოლოდ ბუნებრივი აირი, რომელსაც ჩვეულებრივი დეტექტორი ვერ ხედავს. შესაბამისად, არსებობს მკაფიო საჭიროება ისეთი პრევენციული სისტემისა, რომელიც საფრთხეს რეალურ დროში ამოიცნობს.

05 — ალგორითმი

ამოცნობის ლოგიკა

სისტემა მუშაობს ოთხ მდგომარეობიან მანქანაზე (state machine), FreeRTOS-ის პარალელურ ნაკადებზე.

01

IDLE

უმოქმედობა. სისტემა ელოდება ღილაკის დაჭერას მონიტორინგის დასაწყებად.

02

MONITORING

აქტიური სკანირება. როცა მაქს. ტემპერატურა ≥ 80°C — გადადის ცეცხლის რეჟიმში.

03

FIRE ACTIVE

ცეცხლი დაფიქსირებულია. სისტემა აკვირდება ვარდნას ორი პარალელური მეთოდით.

04

ALARM

განგაში: ბაზერი + Telegram, სანამ მომხმარებელი ღილაკს ხელახლა არ დააჭერს.

მეთოდი 1 მკვეთრი ვარდნა

თუ ტემპერატურა ჩამოსცდება 55°C-ს ზედიზედ 8 კადრის (≈4 წმ) განმავლობაში — განგაში. ეფექტურია გაზის სანთურაზე, რომელიც სითბოს არ ინარჩუნებს.

მეთოდი 2 გაგრილების ტენდენცია

თუ ტემპერატურა ეცემა ≥ 0.2°C/კადრი ზედიზედ 10 კადრის (≈5 წმ) განმავლობაში — განგაში. ეფექტურია სქელკედლიანი ჭურჭლისთვის (ქვაბი, ტაფა), რომელიც დიდხანს ინარჩუნებს სითბოს.

ფსევდოკოდი — FIRE_ACTIVE მდგომარეობა 
CASE FIRE_ACTIVE:
  peakTemp    ← max(peakTemp, maxTemp)
  delta       ← maxTemp − previousTemp

  // მეთოდი 2 — გაგრილების ტენდენცია
  IF delta ≤ −0.2°C THEN coolingFrames++
  ELSE coolingFrames ← 0
  previousTemp ← maxTemp

  // მეთოდი 1 — მკვეთრი ვარდნა ზღვარს ქვემოთ
  IF maxTemp < 55°C THEN
    coldFrameCount++
    IF coldFrameCount ≥ 8 THEN Raise_Alarm()
  ELSE coldFrameCount ← 0

  IF coolingFrames ≥ 10 THEN Raise_Alarm()

07 — პროტოტიპი

მოწყობილობა

კორპუსი დამზადებულია 3D ბეჭდვით (Fusion 360). წინა პანელზე — თერმული კამერა, ზედა პანელზე — ღილაკი და ხმის ღიობი.

ვიდეო დემონსტრაცია

პირველადი გაშვება

  1. შეაერთეთ მოწყობილობა კვების წყაროსთან.
  2. განათავსეთ გაზქურის გვერდით ისე, რომ კამერა ხედავდეს ოთხივე სანთურას.
  3. ჩართვისთანავე იქმნება Wi-Fi წერტილი ThermalSystemSetup.
  4. დაუკავშირდით ამ ქსელს — ავტომატურად გაიხსნება კონფიგურაციის გვერდი.
  5. შეიყვანეთ Wi-Fi მონაცემები და Telegram Chat ID, შეინახეთ.
  6. ღილაკზე დაჭერით ჩართეთ მონიტორინგი.
